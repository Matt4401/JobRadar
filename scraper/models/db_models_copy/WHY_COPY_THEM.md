### The goal of this directory is to have a copy of the models that are used in the database. This is useful for a few reasons:

1. **Version Control**: Having a copy of the models allows developers to track changes over time. If a model is modified in the main codebase, having a copy ensures that previous versions are still accessible for reference or rollback.
2. **Testing and Experimentation**: Developers can experiment with changes to the models without affecting the main codebase. This is particularly useful for testing new features or optimizations.
3. **Documentation**: The copied models can serve as a form of documentation, providing insight into the structure and relationships of the database models. This can be helpful for new developers joining the project or for anyone needing to understand the data schema without diving into the main codebase.

> For simplify, its a way better to have a JSON copy of this models, instead of having them duplicates.
